Managing Common Standard Platform

IT Analyst, Project Leader

Background

The customer implemented a common desktop platform in 3 locations of the enterprise. The common platform follows a special standard concept aund contains software packages adapted for automatic distribution to office computers. Those packages guarantee a version controlled and easy deployment of applications. The framework required maintanance and a well functioning communication between stakeholders. The concept called for a standardized handlingn of client- and server installation with their respective software packages. In order to establish a controlled handling, including Life Cycle Management of software and standardized setup of the current and future infrastructure, this analysis was initiated. Improvement of communication between internal and external stakeholders, defined responsibilities and high data security required improvement.

Scope of transformation

Questions like:
  • Who is owner of the process?
  • How should software packages be managed within the company?
  • How does the software packaging process work within the company?
  • Is there a suitable technical infrastructure available for piloting and testing?
  • Who is managing the common desktop platform in future?
  • How is Lifecycle Management and conceptional changes handled in future?
  • Who are the decision makers?
had to be answered.

Scope of my deliverables

  • Define roles & responsibilities for software packaging and respective roles in the process.
  • Define tasks for each role
  • Setup communication channels and working processes between stakeholders
  • Define criterias for when to create a software package based on usage, costs and importance of the package
  • Identify the prerequisites and skills for managing the implemented standard environment
  • Draw borderlines to related processes
  • Document anlaysis, recommendations and results in a common document

The analysis was documented in a final report presented to the Steering Group.
