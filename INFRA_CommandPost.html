Command Post & Patrol - early warning & monitoring tool

System Manger, Infrastructure Architect

Background

Patrol is a systems, applications, and event management tool for database and system administrators. It provides an environment where it is possible to monitor the status of every vital resource in the managed distributed environment. CommandPost/Patrol borderlines for monitoring at customer sites globally, is to reflect whether or not a system is available, has decreased availability or is not available at all. CommandPost should alert on incidents that may cause unavailability but not do performance monitoring, disk monitoring in a manner that enables for capacity management.

Scope of transformation

Since the customer had to implement the tool on a global base across different operating systems, workgroups were established for each OS to come forward with a suitable solution on respective OS. The software consisted of different agents to be installed on different platforms. The requirement to package and prepare the software for global automated distribution was included in the scope.

Scope of my deliverables

  • Requirements analyses
  • Specification of requirements
  • Configuration and testing of the software on Windows NT Servers
  • Prepare the agent for global rollout on Windows NT Servers
