System upgrade Windows NT 3.51 to Windows NT 4.0

IT Coordinator, System Manager

Background

With beginning of 1997 the Swedish central IT organisation developed a common desktop concept valid for software development machines. This should guarantee that inhouse application development was performed on standardized computers. Inhouse developed applications were to be distributed to many offices. A standard setup should assure an easy maintance circle for IT operations. One of the early adopters of this standard system was the Austrian IT organisation, implementing this concept in all countries supported by them. In order to fullfill the to-date requirements systems needed to be upgraded from Windows 3.51 to Windows 4.0 on the server-side and from Windows 3.11 to Windows 95 on client-side.

Scope of transformation

A common project was established which should help co-workers and developers to utilize the first global available edition of a common server- and desktop platform. Installation checklists, creating of distribution packages, technical adaptions for Central European Countries and their specific codepages including scripts supporting special setups had to be prepared. Challenges occured specifically in the areas of network setup. Austria and its supported Central European Countries was running on the Tokenring Topology where the rest of the customers units was using Ethernet as network standard. Technical adaptions to the automated installation routines had to be done in Austria. The success of technical adaptions were crucial for the decision to use the standard setup in those countries.

Scope of my deliverables

  • Coordination with country IT, planning and responsible for technical execution
  • Planning and preparation pilot countries
  • Test and supervise initial common desktop tools
  • Integration and adaption of prerequisites to the Tokenring environment
  • Adaption of east character settings for Win95 and Office95
  • Training/education for local IT personel/System Managers in respective countries
  • create software packages for automated distribution
  • Extended User Support for troubleshooting post installation issues
  • Handover to local IT
  • System Exchange and technical guidance from Windows 3.51 to Windows NT 4.0
