Feasibility Study for Windows 7 & Windows 2008R2

IT Analyst

Background

The standard platform IKEA Common Client 3 (ICC3) was the main server/client infrastructure setup within Inter IKEA Systems. Its basic functions contribute to automated server/client installations and simplify the maintenance, life cycle management and reports with help of a single tool. ICC3 is a product, developed and supported by IKEA IT, built on Microsoft technology containing Windows XP and Windows 2003 Server.IKEA IT has created a new version of the ICC platform based on Windows Server 2008R2 64bit and Windows7 64bit. One of the main goals of this new version is to make a better working day for the many users, which is achieved by providing a stable,secure, cost effective and common desktop environment. The objective is to deliver a pre-study that provides sufficient information for taking a decision how to implement the next version of ICC and MS Office.

Scope of transformation

  • Description and analysis of implementation scenarios, estimates on related costs.
  • Outline Microsoft Office dependencies on other Microsoft Products such as Exchange and Office Communicator.
  • Description and analysis of future operational costs in relation to maintanance, software licences and labour.
  • Description of benefits and disadvantages by implementing the updated platform.
  • Identification of possible costs and risks of not implementing the updated platform
  • Recommendation for next steps to the steering group

Scope of my deliverables

The pre-study was devided into three parts. Part I should contain solution alternatives, e.g. continuing with the current setup or implementing the updated platform. Part II contained implementation alternatives. Part III was to create the summary report as deliverable.

Deliverables for Part I (solution alternatives) in detail:

  • Overview of current concept. The project will draw a picture of the current ICC3 environment. Information and data will be collected in project specific meetings.
  • At least two alternatives will be presented in this deliverable. The alternatives will be worked out in project meetings together with the project team/reference persons.
  • Benefits & Disadvantages of each alternative: For each alternative the project team will gather benefits & disadvantages, including security risks, changes in core modules (software), education, IT efficiency, End user benefits.
  • Costs of each alternative including operational costs: the project team will gather all costs arising from the chosen alternatives, including TCO.

Deliverables for Part II (implementation alternatives) in detail:

  • Description of different implementation alternatives. At least two implementation scenarios will be developed within the project team.
  • Benefits & Disadvantages of each alternative. For each implementation scenario benefits and & disadvantages will be opposed and documented.
  • All costs including training, architecture changes and software changes/modules adaptations will be calculated and documented.

The anlaysis was summarized and documented in a final report presented to the Steering Group
