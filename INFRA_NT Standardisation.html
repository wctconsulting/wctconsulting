Windows NT Standardisation & Consolidation

System Manger, Infrastructure Architect

Background

Stabilizing and securing the Windows NT and Terminal server platform within customer premises to reach the goal "One IT for one IKEA" was the trigger for the project "Windows NT Standardisation and consolidation". Investments in Windows NT Servers were done differently worldwide. Every organisation purchased servers from different suppliers locally with specific country agreements. This resulted in higher costs for each server installation, non-standardized setup and lack of input to global service level agreements within global IT departments and the local offices and stores. Applications developed inhouse for Windows NT Servers could not easily be secured for operational functionality and stability as there were too many differnt hardware platforms existing. This fact made it difficult to develop applications globally.


The following objectives have been defined, being a solid outcome of the project.

  • Consolidate and unite the IT Infrastructure
  • Maintain a stable and secure operation support
  • Standardized routines for development when implementing new applications
  • Simple and rational off the shelf solutions

Scope of transformation

The project was initiated by the investigation of the best ways on how to setup a global standardized NT Server environment. Different workshops and brainstorming sessions led to defining common system management routines for operations based on the common knowledge of key IT personal globally. Evaluation of hardware vendors, functional distinctions between servers,description of common agreements between the customer, countries and suppliers as well as categorizing the impact on the organisational structure defined a global working structure which is still in use to date.

Scope of my deliverables

  • Investigate in how to standardize a global server infrastructure
  • Evaluate and suggest possible hardware vendors
  • Investigate in how servers can be divided into different functions servers isolated from each other (for example: File/Print server standard, Database standard, application standard, Web application Standard, Communication process standard
  • Drafting of guidelines on how to setup clustered NT environment
  • General standardisation proposal for HomeDirectory handling, shares & permission setup
  • Draft recommendation on general Service Level Agreements on all levels (central handling, local handling)
  • Evaluation of ownership of servers
